# 用 TypeDown 打开 md 文件越来越慢，于是自己搓了一个 Markdown 阅读器

> 结果一不小心，搓出了一个完整的应用。

---

事情要从一句吐槽说起。

我用 TypeDown 打开 Markdown 文件，越来越慢。点一下，等三秒。再点一下，再等三秒。

去官网一看，免费版不维护了。

合着这工具就是个套壳浏览器，每次启动都要把整个 Chromium 拉起来，难怪慢。安装包一百多兆，打开一个几百 KB 的笔记，要等半天。

我就想：**Markdown 渲染，本身又不是什么高深的东西。它就是文本转 HTML。我为什么不能自己做一个？**

于是开搓。

名字想好了，叫**好记**。因为我还有一个项目叫"好找"——一个文件管理系统。一个帮你找到文件，一个帮你记下想法，正好凑一对。

---

## 第一步：先让它能"看"

最朴素的需求：把 .md 文件拖进去，能看到渲染后的样子。

这一步其实不难。Markdown 解析有现成的库（markdown-it），代码高亮有 highlight.js，数学公式有 KaTeX，流程图有 Mermaid。把它们拼起来，一个下午就跑通了。

拖入文件 → 解析 → 渲染。标题、表格、代码块、公式、流程图，全都出来了。

但这里有个小插曲：第一次跑起来，数学公式全是乱码，显示成 `KATEX0`、`KATEX1` 这样的裸字符。

排查了半天，发现是我用 `\u0000`（NUL 字符）做占位符，结果 markdown-it 在规范化阶段把 NUL 替换成了别的字符，导致还原时匹配不上。换成纯 ASCII 的 `@@KATEX@@` 占位符就好了。

**造轮子的乐趣，一半在这些 debug 的瞬间。**

---

## 第二步：要能"改"

光能看不够，还得能编辑。

一开始我做的是分屏——左边写，右边渲染。这是很多 Markdown 编辑器的做法。

但用着用着，发现问题：**鼠标在左边编辑器里的位置，和右边渲染出来的位置对不上。**

我滚到编辑器的第 100 行，右边的预览却停在第 80 行的内容上。因为源码一行对应渲染后可能好几行（一行 Mermaid 代码渲染成 400px 高的图），按比例同步根本对不齐。

这是几乎所有分屏 Markdown 编辑器的通病。

然后我想起 Obsidian——它不是分屏，是**实时渲染**。光标所在的行显示源码，其余行自动渲染成最终样子。点进哪行，那行就"褪"回源码；移走光标，又渲染回去。

**这才是对的体验。** 于是砍掉分屏，重做。

---

## 最难的一关：点进代码块编辑

实时渲染听起来美好，做起来要命。

最难的是**代码块、表格、公式、流程图**这些"块级元素"。它们是多行的，我希望它们默认渲染成漂亮的成品（高亮代码、真表格、公式图、流程图），但点进去又能编辑。

第一版我用 CodeMirror 的 block widget 把整块替换成渲染后的样子。结果——**根本点不进去**。

查了半天才发现，这是 CodeMirror 的硬限制：block widget 替换的范围，光标无法通过点击进入。死锁了：点 widget → 光标停在块外 → 检测"光标不在块内" → 永远渲染 → 永远进不去。

后来想通了破局点：**widget 不该被动等光标进来，而是应该在被点击时，主动把光标推进块里。**

```
点击 widget
  ↓ widget 拦截 mousedown
主动 dispatch 光标到块起始位置
  ↓ 触发选区变化
装饰层检测到光标进入块范围
  ↓ 撤掉该块的替换装饰
源码显现，光标已在源码中 → 可编辑
```

这一帧的切换发生在同一个更新周期里，用户感觉不到延迟。体验就和 Obsidian 一样——点哪进哪。

这个机制打通后，代码块、表格、公式、Mermaid 流程图全都能点进即编辑了。

**说实话，这是整个项目最有成就感的瞬间。**

---

## 第三步：让它像个产品

核心功能跑通后，开始往"产品"上靠：

- 🌙 **暗色模式**——跟随系统，连代码高亮主题都跟着切
- 📑 **大纲导航**——左侧自动生成目录，点击跳转，自动高亮当前章节
- ⌨️ **快捷键**——Ctrl+B 加粗、Ctrl+I 斜体、Ctrl+E 代码、Ctrl+S 保存
- 🔤 **字号调节**、🕘 **最近文件**

这些都还好，按部就班做出来。

---

## 最后一步：让它"秒开"

做到这里，好记已经是一个功能完整的网页版编辑器。但我最初的痛点——**打开太慢**——还没真正解决。

网页版受浏览器限制，打开 .md 文件本身就有开销。要真正秒开，必须做成**桌面应用**。

这里有两个选择：

- **Electron**：和 TypeDown 一样的方案。打包一个大 Chromium。慢、大、我不想要。
- **Tauri**：用系统自带的 WebView，Rust 做后端。小、快、现代。

选了 Tauri。

过程中踩了几个坑：Rust 工具链的 PATH 没配好、MSVC 链接器、中文产品名导致 MSI 打包失败（改用 NSIS 解决）……但最终跑通了。

结果：

| | TypeDown (Electron) | **好记 (Tauri)** |
|---|---|---|
| 安装包 | 100+ MB | **4.9 MB** |
| 打开速度 | 慢 | **秒开** |
| 维护 | 免费版停更 | **自己掌控** |

体积只有原来的 **二十分之一**。

最关键的是——**.md 文件双击，秒开**。这是我一开始就想要的东西。

---

## 一些细节

好记现在的样子：

- 应用图标是一根蓝色羽毛（轻、写）
- 关联的 .md 文件图标是另一个吉祥物（和应用图标区分开）
- 安装后，.md / .markdown / .mdx / .mdown 默认用好记打开
- Ctrl+S 直接写回原文件（不再是浏览器下载）

技术栈：React + CodeMirror 6 + Tauri 2，渲染核心是 markdown-it + highlight.js + KaTeX + Mermaid。

## 写在最后

这事的起因很小——就是一个工具太慢了，不维护了。

但顺着这个不满往下做，我得到了：

1. 一个**真正属于自己的工具**，想怎么改就怎么改
2. 搞懂了 Obsidian 实时渲染背后的机制
3. 完整走了一遍"网页原型 → 桌面应用"的全流程
4. 一个 4.9MB、秒开的 Markdown 编辑器

**很多时候，"造轮子"不是浪费时间。它是你深度理解一个东西的最快路径。**

下次再遇到一个不顺手的工具，与其忍着，不如想想：它到底做了什么让我不爽？这个东西自己做一个，难吗？

也许，下一个周末就有事干了。
